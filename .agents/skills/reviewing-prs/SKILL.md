# Reviewing PRs

## Steps

1. Read the full diff
2. Check for security issues (OWASP top 10)
3. Check test coverage for changed code paths
4. Verify naming conventions and code style match the repo

## Gotchas

- Don't approve PRs that add dependencies without justification
